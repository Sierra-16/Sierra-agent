# 架构图

## 流程

1. 从代码或用户描述中提取组件、边界、数据流、协议和信任关系。
2. 信息不足且会改变图结构时，调用 `request_user_input`。
3. 先设计分层和连接，再生成图；连接线先绘制，节点后绘制，避免箭头穿过节点文字。
4. 读取 `references/svg-layout.md`，按其中的坐标与连线检查表组织 SVG。
5. 使用 `skill_render_template` 渲染 `templates/architecture.html`，再用 `write_file` 保存结果。
   `diagram_svg` 填入完整的内联 SVG，`notes` 填入简短的关系说明列表。
6. CSS 与 SVG 全部内联，不依赖网络字体或 JavaScript。
7. 检查 SVG `viewBox`、画布尺寸、边缘留白和小屏可读性。

## 视觉语义

- 背景：`#0b0f0d`
- 前端/客户端：青色 `#22d3ee`
- 服务/智能体：绿色 `#34d399`
- 数据库/记忆：紫色 `#a78bfa`
- 外部系统/MCP：金色 `#fbbf24`
- 安全与风险边界：玫红 `#fb7185`
- 辅助文字：`#94a3b8`

使用颜色表达类型，而不是装饰。边界用虚线矩形，异步流使用虚线箭头，主请求流使用实线箭头。每个节点包含短名称和一行职责，详细说明放在图下方，不塞进 SVG。

## 输出要求

- 默认保存为当前工作区的 `<主题>-architecture.html`。
- 使用稳定的网格、最小 32px 节点间距和唯一 marker ID。
- 图例放在所有边界之外。
- 不伪造尚未确认的服务或连接；推断项使用 `?` 或备注说明。
- 保存后报告文件路径和图中表达的关键关系。
