# 深度研究

多源搜索、交叉验证、综合成一份有引用的报告。

**核心原则:** 单一来源不可信。交叉验证是研究的底线。

## 适用场景

- 用户想要全面了解一个话题
- "帮我调研 X"、"写一份关于 Y 的报告"
- 技术选型、竞品分析、行业调研
- 跳过: 简单事实查询（直接搜索就够了）

## 流程

### 第一步 — 拆解问题

把主问题拆成 2-4 个子问题，每个子问题独立可搜索。

```
用户: "调研一下 Python 异步框架"
子问题:
1. Python 主要异步框架有哪些？
2. 各自的性能对比和适用场景？
3. 社区活跃度和维护状态？
4. 生产环境使用案例？
```

### 第二步 — 多源搜索

每个子问题用 `web_search` 搜索，至少用 2 组不同关键词：

```
web_search("Python 异步框架 对比 2024")
web_search("Python async framework performance benchmark")
```

记录每个搜索结果的：
- 来源 URL
- 标题
- 关键数据/观点

### 第三步 — 交叉验证

对每个关键信息点，用 `web_fetch` 抓取至少 2 个不同来源验证：

```
web_fetch("来源A的URL")  → 数据 X
web_fetch("来源B的URL")  → 数据 X 是否一致？
```

不一致 → 注明分歧，继续搜索第三个来源。
一致 → 确认，标记为"已验证"。

### 第四步 — 综合报告

按以下结构输出：

```markdown
# [主题] 调研报告

## 概述
一段话总结核心发现。

## 详细分析

### 子问题 1: ...
- 发现 1 [来源1](URL) [来源2](URL)
- 发现 2 [来源](URL)

### 子问题 2: ...
...

## 交叉验证结果
| 信息 | 来源1 | 来源2 | 一致？ |
|------|------|------|--------|
| ...  | ...  | ...  | ✅/❌   |

## 结论
- 核心结论 1
- 核心结论 2

## 来源列表
1. [标题](URL)
2. ...
```

### 第五步 — 限定和自知

在报告末尾注明：
- 哪些信息已验证（2+ 来源）
- 哪些信息是单一来源（需要进一步确认）
- 哪些是本模型的推断（非来源信息）

## 质量准则

- 每个关键发现至少 2 个独立来源
- 优先原始来源而非转载
- 标注发布日期，超过 1 年的注明
- 不编造来源，找不到就说找不到
- 中文和英文来源都搜
