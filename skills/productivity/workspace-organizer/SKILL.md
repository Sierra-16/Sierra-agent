# 工作区整理

## 安全流程

1. 使用 `list_directory`、`search_files` 和只读命令建立目录清单。
2. 明确目标范围、保留规则和“完成后应该长什么样”。范围不清时调用 `request_user_input`。
3. 对多步骤整理调用 `update_plan`。
4. 生成变更预览：源路径、目标路径、操作类型、冲突和预计数量。
5. 先执行一个小批次并验证，再继续剩余项目。
6. 每次删除、覆盖、批量移动或重命名都使用 `powershell`，接受 Sierra 权限确认。
7. 完成后重新列出目标目录，并报告成功、跳过和失败项。

## Windows 与路径规则

- 使用 PowerShell 原生命令和 `-LiteralPath`，不要把路径拼接成跨 shell 删除命令。
- 递归移动或删除前解析绝对路径，确认所有目标仍在当前工作区或用户明确指定的目录内。
- 处理 Windows 大小写不敏感、保留名称、路径空格和 OneDrive 同步冲突。
- 目标已存在时停止并列出冲突，不自动覆盖。

## 默认保护

除非用户明确要求，不整理或删除：

- `.git/`、`.env`、凭据和配置密钥
- `.venv/`、`node_modules/` 与包缓存
- `memory/`、`conversations/`、`tasks/`、`logs/`
- 用户未提交的源代码和未知二进制文件

生成目录如 `dist/`、`build/`、`__pycache__/` 也要先确认项目当前没有依赖它们运行。

## 删除与归档

- 能归档时优先归档，只有用户明确要求清理才删除。
- 不用“写成空文件”代替删除，也不用移动到未知临时目录假装完成。
- 文件是否重复优先比较哈希，不只比较名称或大小。
- 操作被拒绝或部分失败时保留原状态，并准确汇报。
