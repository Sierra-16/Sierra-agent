# 长期记忆整理

## 记忆类型

- `target="user"`：用户稳定身份、偏好、能力、协作方式和明确要求记住的信息。
- `target="memory"`：项目架构、长期决策、已确认状态和持续待办。
- 向量对话记忆由系统自动同步；不要使用 PowerShell 或 SQLite 命令直接修改数据库。

## 整理流程

1. 在得到读取许可后查看 `memory/USER.md` 与 `memory/MEMORY.md`，或者使用当前界面提供的记忆状态。
2. 将条目分为：保留、合并、更新、删除候选。
3. 判断是否值得长期保存：未来会改变 Sierra 的回应或行动才保留。
4. 新信息补充旧信息时形成一条完整表述，不并排保留多个近义版本。
5. 出现冲突时优先采用用户最新、明确的纠正；时间和含义不清楚时调用 `request_user_input`。
6. 使用 `save_memory` 新增或保存合并结果。
7. 删除前展示将被删除的内容，确认关键词只匹配目标条目，再使用 `delete_memory`。

## 不应保存

- API key、token、密码、证件和精确住址。
- 暂时情绪、普通闲聊、一次性任务和搜索结果细节。
- 助手的猜测、未确认的人格推断和第三方隐私。
- 完整代码、长日志和可以从仓库重新读取的内容。

## 冲突处理

- “用户明确纠正”高于旧记忆。
- “较新”不自动等于“正确”；区分临时变化与稳定变化。
- 删除可能匹配多条记录时不要执行，先换成唯一短语或让用户选择。
- 用户可以要求不保存某类信息；这种边界本身可以作为稳定协作偏好保存。

## 交付

说明新增、更新、合并和删除了什么。没有值得修改的内容时明确说明，不为了显得有成果而制造记忆。
