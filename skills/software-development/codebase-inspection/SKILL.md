# 代码库检查

## 流程

1. 确认工作区和分析范围，避免扫描工作区之外的目录。
2. 使用 `list_directory` 和 `search_files` 找入口、配置、测试、构建文件与核心包。
3. 需要快速、可重复的文件清单统计时，调用 `skill_run_script` 执行
   `scripts/codebase_inventory.py`。脚本参数可传工作区内的相对子目录，默认 `.`。排除：
   `.git`、`node_modules`、`.venv`、`venv`、`dist`、`build`、缓存和生成目录。
4. 需要 LOC 或语言统计时，先检查 `pygount` 是否已安装。存在时使用 summary/JSON 输出；不存在时用文件后缀和逐文件行数做近似统计，不自动安装依赖。
5. 抽样阅读关键文件，验证目录名称表达的职责是否真实。
6. 输出架构、执行入口、依赖方向、测试布局和主要风险。

## 报告要求

- 区分“工具统计”“从代码确认”和“推断”。
- LOC 只表示规模，不把行数当作质量或复杂度结论。
- 对大型仓库先做顶层地图，再按用户问题深入目标模块。
- 给出真实文件路径；没有证据时明确写未知。

## 验证

- 依赖和生成目录已排除。
- 至少读取一个入口、一个配置和一个测试文件。
- 总文件数与各语言文件数没有明显矛盾。

本技能根据 Hermes `codebase-inspection` 工作流改写，命令与安全边界适配 Sierra。
