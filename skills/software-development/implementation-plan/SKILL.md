# 实现计划

## 边界

- 仅在用户明确要求“先规划”“不要写代码”“给方案”时使用。
- 不调用会修改文件或外部状态的工具。
- 普通复杂任务不使用本技能；调用 `update_plan` 管理进度并继续执行。

## 流程

1. 使用只读工具了解代码结构、现有模式和约束。
2. 找出需要改变的模块、接口、数据流和测试面。
3. 遇到会实质改变方案的缺失信息时调用 `request_user_input`。
4. 将工作拆成可以独立验证的小步骤。
5. 为每一步写明文件位置、行为变化、验证方法和依赖关系。
6. 标出风险、迁移策略以及仍未确认的假设。

## 输出

按以下顺序组织：

1. 目标与范围
2. 当前实现概况
3. 实施步骤
4. 测试与验收
5. 风险和待确认项

不要伪造尚未读取的文件、函数或测试结果。用户没有要求保存时，直接在对话中交付计划。
